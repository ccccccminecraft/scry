# 16: デッキバージョン一括適用

## 概要

デッキ管理画面から、フィルター条件に合致する複数の対戦履歴に対して、選択したデッキバージョンを一括で紐づける機能。

既存の機能（15: 対戦詳細画面での1件ずつの手動紐づけ）を補完するもので、過去の対戦履歴に遡ってデッキバージョンをまとめて適用する際に使用する。

---

## 操作フロー

1. デッキ管理画面でデッキを選択する
2. バージョン一覧の対象バージョン行にある「一括適用」ボタンをクリックする
3. モーダルが開く
4. フィルター条件（プレイヤー・フォーマット・デッキ名・期間）を指定する
5. 対象件数が表示される
6. 「適用する」ボタンをクリックして確定する

## API

### POST /api/matches/bulk-assign-deck-version

指定したフィルター条件に合致する `match_players` の `deck_version_id` を一括更新する。

**リクエストボディ**

```json
{
  "deck_version_id": 3,
  "player": "necepanecepa12",
  "format": "modern",
  "deck_name": "Amulet Titan",
  "date_from": "2024-01-01",
  "date_to": "2024-03-31"
}
```

| フィールド | 型 | 必須 | 説明 |
|------------|-----|------|------|
| `deck_version_id` | integer | ✅ | 適用するデッキバージョン ID |
| `player` | string | ✅ | 対象プレイヤー名 |
| `format` | string | - | フォーマット絞り込み |
| `deck_name` | string | - | デッキ定義名による絞り込み（`match_players.deck_name`） |
| `date_from` | string (YYYY-MM-DD) | - | 対戦日の開始（以降） |
| `date_to` | string (YYYY-MM-DD) | - | 対戦日の終了（以前） |

**レスポンス**

```json
{ "updated": 12 }
```

### GET /api/matches/bulk-assign-deck-version/count

上記と同じフィルター条件で対象件数のみを返す（適用前のプレビュー用）。

**クエリパラメータ**: リクエストボディと同じフィールドをクエリパラメータとして受け取る。

**レスポンス**

```json
{ "count": 12 }
```

---

## 処理フロー（バックエンド）

```
1. deck_version_id の存在確認 → なければ 404
2. player を条件に match_players を JOIN して絞り込み
   - format が指定されていれば Match.format で絞り込み
   - deck_name が指定されていれば match_players.deck_name で絞り込み
   - date_from / date_to が指定されていれば Match.played_at で絞り込み
3. 条件に合致する match_players.deck_version_id を一括 UPDATE
4. 更新件数を返す
```

## UI 設計

### DeckBuilderView — バージョン行への追加

バージョン一覧の各行に「一括適用」ボタンを追加する。

```
[ v1  2024-01-15  手動入力 ]  [ 削除 ] [ 一括適用 ]
[ v2  2024-03-01  .dek     ]  [ 削除 ] [ 一括適用 ]
```

### 一括適用モーダル

```
[ "Amulet Titan v2" を対戦履歴に一括適用 ]

プレイヤー  [ necepanecepa12 ▼ ]  ← playerList から選択
フォーマット [ Modern ▼ ]
デッキ名    [ Amulet Titan    ]   ← 選択したデッキ名で自動補完・編集可
期間（開始）[ 2024-01-01 ]
期間（終了）[ 2024-03-31 ]

対象: 12 件                      ← フィルター変更のたびに再取得

[ キャンセル ]  [ 適用する ]
```

- プレイヤーはすでに設定済みの `default_player` を初期値とする
- デッキ名は親デッキの `name` で自動補完するが編集可能にする
- フィルター変更のたびに count API を呼び出し対象件数をリアルタイム更新する
- 対象が 0 件の場合は「適用する」ボタンを無効化する
- 確定後は「N 件に適用しました」トーストを表示する

## エラーハンドリング

| 状況 | 対応 |
|------|------|
| 存在しない `deck_version_id` | 404 |
| `player` 未指定 | 400 |
| 対象 0 件で適用ボタン押下 | ボタンを disabled にして防止 |
| 通信エラー | エラートーストを表示 |

## 動作確認手順

1. デッキ管理画面でデッキ・バージョンを選択する
2. バージョン行の「一括適用」ボタンをクリックしてモーダルが開くことを確認する
3. フィルターを変更するたびに対象件数が更新されることを確認する
4. 「適用する」でトーストが表示され、対戦詳細画面で該当対戦のデッキバージョンが反映されていることを確認する
5. 対象 0 件のとき「適用する」ボタンが無効になることを確認する
