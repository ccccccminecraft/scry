# Scry 機能実装計画

## 進め方

各機能を以下のサイクルで実装する。

```
1. 詳細設計ドキュメント作成（docs/impl/）
2. レビュー・合意
3. 実装
4. 動作確認
```

詳細設計ドキュメントには以下を含める：

- **対象ファイル一覧**: 新規作成・編集するファイル
- **インターフェース定義**: 関数シグネチャ・型・データ構造
- **処理フロー**: 主要な処理の順序・分岐
- **エラーハンドリング**: 想定するエラーと対処
- **動作確認手順**: 実装完了を確認する方法
