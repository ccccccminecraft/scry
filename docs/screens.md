# 画面設計

## 画面一覧

| 画面ID | 画面名 | 概要 |
|--------|--------|------|
| S01 | ホーム | アプリ起動時のトップ画面。バックエンド接続確認 |
| S02 | インポート | .dat ファイルの取り込み画面 |
| S03 | 対戦履歴一覧 | インポート済みの対戦ログ一覧 |
| S04 | 対戦詳細 | 1マッチの詳細（ゲームごとの情報） |
| S05 | 統計ダッシュボード | 集計統計・グラフ表示 |
| S06 | AI 分析 | AI によるプレイパターン分析結果 |
| S07 | 設定 | LLM API キーの登録・管理 |
| S08 | 分析プリセット管理 | プロンプトテンプレート・質問セットの管理 |

---

### S01 ホーム

**目的**: アプリの起動確認・バックエンド接続ステータスの表示

**レイアウト**

```
┌────────────────────────────────────────────────┐
│  Scry                                          │
│                                                │
│      ✅ Backend connected                      │
│      （または ❌ Backend not connected）        │
│                                                │
│      [ログをインポートする]  [履歴を見る]        │
│                                                │
└────────────────────────────────────────────────┘
```

**仕様**

- 画面表示時に `GET /api/health` を呼び出す
- 成功時: 「✅ Backend connected」を緑色で表示
- 失敗時: 「❌ Backend not connected」を赤色で表示

### S02 インポート

**目的**: .dat ファイルを選択してDBに取り込む

**レイアウト（Idle 状態）**

```
┌────────────────────────────────────────────────┐
│  ログのインポート                               │
│                                                │
│  ┌──────────────────────────────────────────┐  │
│  │  ファイルをドラッグ＆ドロップ             │  │
│  │  または [ファイルを選択]                  │  │
│  └──────────────────────────────────────────┘  │
│                                                │
│  ────────── または ──────────                  │
│                                                │
│  [📁 フォルダから一括取り込む]                  │
│  ヒント: C:\Users\[ユーザー名]\AppData\Local\  │
│          Apps\2.0                              │
│                                                │
└────────────────────────────────────────────────┘
```

**レイアウト（ScanResult 状態）**

```
┌────────────────────────────────────────────────┐
│  ログのインポート  ← 戻る                       │
│                                                │
│  📁 C:\Users\...\Apps\2.0                      │

│  [すべて選択]                                  │

│  ☐ Match_GameLog_7b3e9d04-...dat  2024/01/13  │
│    （インポート済み）                           │
│  ...                                           │
│                                                │
│                    [選択した17件をインポート]   │
│                                                │
└────────────────────────────────────────────────┘
```

**レイアウト（BatchResult 状態）**

```
┌────────────────────────────────────────────────┐
│  ログのインポート                               │
│                                                │
│  ✅ インポート完了                              │
│                                                │
│  インポート済み   15件                          │
│  スキップ（重複）  2件                          │

│                                                │
└────────────────────────────────────────────────┘
```

**仕様**

- 単体: Electron の `dialog.showOpenDialog` でファイル選択 / ドラッグ＆ドロップ対応
- フォルダスキャン: `dialog.showOpenDialog({ properties: ['openDirectory'] })` でフォルダ選択
- スキャン対象パターン: `Match_GameLog_*.dat`（サブディレクトリ再帰検索）
- フォルダ選択ダイアログの初期パス: `C:\Users\[ユーザー名]\AppData\Local\Apps\2.0`（ヒント）
- スキャン結果ではインポート済みファイルをグレーアウト・デフォルト未選択で表示
- 一括インポート中はプログレスバーで進捗を表示（N / M 件）
- `POST /api/import/batch` で一括送信

### S04 対戦詳細

**目的**: 1マッチの詳細情報を表示する

**レイアウト**

```
┌────────────────────────────────────────────────┐
│  ← 戻る                                        │
│  対戦詳細 2024/01/15 10:30   フォーマット: modern │
│  PlayerA vs PlayerB   → PlayerA 勝利            │
│                                                │
│  PlayerA  デッキ: [Modern Burn      ] [✎]      │
│           ゲームプラン: [aggro ▼]              │
│  PlayerB  デッキ: [未設定           ] [✎]      │
│           ゲームプラン: [-- ▼]                 │
│                                                │
│  ゲーム 1  先手: PlayerA  ターン数: 8           │
│    PlayerA: マリガン 0回 / PlayerB: マリガン 1回 │
│    [アクションログを見る ▼]                     │
│                                                │
│  ゲーム 2  先手: PlayerB  ターン数: 12          │
│    PlayerA: マリガン 0回 / PlayerB: マリガン 0回 │
│    [アクションログを見る ▼]                     │
│                                                │
│  ゲーム 3  先手: PlayerA  ターン数: 6           │
│    PlayerA: マリガン 1回 / PlayerB: マリガン 0回 │
│    [アクションログを見る ▼]                     │
│                                                │
└────────────────────────────────────────────────┘
```

**仕様**

- `GET /api/matches/{match_id}` で詳細取得（`players[].deck_name` / `players[].game_plan` 含む）
- デッキ名はインライン編集（✎ クリックでテキスト入力に変換）
- ゲームプランはドロップダウン選択（`aggro` / `midrange` / `control` / `combo` / 未設定）。未設定時は `null` で扱う
- 変更時に `PATCH /api/matches/{match_id}/players/{player_name}` を呼び出す
- アクションログはアコーディオン形式で展開。展開時に `GET /api/matches/{match_id}/games/{game_id}/actions` を呼び出す（初回のみ取得しフロントにキャッシュ）

### S05 統計ダッシュボード

**目的**: プレイヤーの集計統計・グラフを表示する

**レイアウト**

```
┌────────────────────────────────────────────────┐
│  統計                                          │
│  対象プレイヤー: [PlayerA ▼]                   │
│  対戦相手: [すべて ▼]  デッキ: [すべて ▼]      │
│  ゲームプラン: [すべて ▼]                      │
│                                                │
│  勝率         マリガン率   平均ターン数          │
│  58.3%        31.0%       9.2                  │
│                                                │
│  ┌─────────────────┐  ┌──────────────────┐    │
│  │  勝率推移グラフ  │  │ 先手/後手 勝率比  │    │
│  │  （折れ線）      │  │ （棒グラフ）      │    │
│  └─────────────────┘  └──────────────────┘    │
│                                                │
│  ┌──────────────────┐  ┌─────────────────┐    │
│  │ マリガン回数別    │  │ デッキ別 勝率    │    │
│  │ 勝率（棒グラフ）  │  │ （棒グラフ）     │    │
│  └──────────────────┘  └─────────────────┘    │
│                                                │
│  ┌─────────────────────────────────────────┐  │
│  │  ゲームプラン別 勝率（棒グラフ）          │  │
│  └─────────────────────────────────────────┘  │
│                                                │
│  よく使ったカード TOP20                        │
│  ──────────────────────────────────────        │
│  カード名          使用数  登場ゲーム勝率       │
│  Lightning Bolt    143    64.4%               │
│  Goblin Guide       98    61.2%               │
│  ...                                           │
│                                                │
└────────────────────────────────────────────────┘
```

**仕様**

- **プレイヤードロップダウン**: 画面表示時に `GET /api/stats/players` で選択肢を取得する。取得結果の先頭を自動選択する。プレイヤーが1件もない場合は「対戦ログがありません」を表示してフィルター・グラフ・テーブルを非表示にする。プレイヤー変更時は他のフィルター（対戦相手・デッキ・ゲームプラン）をリセットして統計を再取得する
- `GET /api/stats?player=xxx` で基本統計取得
- 対戦相手ドロップダウン: `GET /api/stats/opponents?player=xxx` で選択肢取得
- デッキ / ゲームプランドロップダウン: `GET /api/stats` レスポンスの `deck_stats` / `game_plan_stats` から選択肢を生成
- フィルタ変更時は `GET /api/stats?player=xxx&opponent=yyy&deck=zzz&game_plan=www` を再取得
- カード統計: `GET /api/stats/cards?player=xxx` で取得（フィルタ非連動、プレイヤー固定）
- グラフライブラリ: ECharts + vue-echarts

### S06 AI 分析

**目的**: AI とチャット形式でプレイパターンについて対話する

**レイアウト（API キー設定済み）**

```
┌────────────────────────────────────────────────┐
│  AI 分析                    対象: [PlayerA ▼]  │
│                   プロンプト: [デフォルト   ▼]  │
│  ─────────────────────────────────────────     │
│  履歴: [マリガンについて... 01/15] [先手/後手...] [新しい会話 +] │
│  ─────────────────────────────────────────     │
│                                                │
│  🤖 PlayerA さんのデータを読み込みました。      │
│     何でも聞いてください！                     │
│                                                │
│  👤 私のプレイの傾向を教えてください            │
│                                                │
│  🤖 先手時の勝率が後手より 8% 高く、           │
│     積極的な展開が得意なスタイルです。          │
│                                                │
│  👤 マリガンについて詳しく教えてください        │
│                                                │
│  🤖 マリガン後の勝率は 38% と...  ▌           │
│     （ストリーミング表示中）                    │
│                                                │
│  ─────────────────────────────────────────     │
│  質問セット: [基本分析 ▼]                      │
│  [私のプレイの傾向を] [先手/後手の勝率の差を]   │
│  [マリガンの影響を]   [改善できるポイントを]    │
│  [直近の勝率推移を]                            │
│  ─────────────────────────────────────────     │
│  [メッセージを入力...]              [送信 ↵]   │
│                 [この会話を削除]  [会話をリセット] │
└────────────────────────────────────────────────┘
```

**レイアウト（API キー未設定）**

```
┌────────────────────────────────────────────────┐
│  AI 分析                                       │
│                                                │
│  ⚠️ AI 分析を利用するには API キーの設定が     │
│     必要です。                                 │
│                                                │
│                         [設定画面を開く →]     │
│                                                │
└────────────────────────────────────────────────┘
```

**仕様**

- 画面表示時に `GET /api/settings` で API キー設定済みか確認
- API キー未設定時: 未設定メッセージと設定画面へのリンクを表示
- プロンプトドロップダウン: `GET /api/prompt-templates` で取得、デフォルトを初期選択
- **セッション履歴バー**: プレイヤー選択時に `GET /api/analysis/sessions?player=xxx` で取得し、タイトル・日付を横並びのタブ形式で表示
  - 「新しい会話 +」ボタンで新規セッションを開始
  - 履歴タブクリックで `GET /api/analysis/sessions/{id}` を取得して過去の会話を表示（読み取り専用）
- プレイヤー・プロンプト変更時は会話をリセットして新規セッションを開始
- **新規セッション開始時**: AI の挨拶メッセージを自動送信。`POST /api/analysis/chat`（`session_id` 省略）を呼び出し、レスポンスの `session_id` をフロントで保持
- **メッセージ送信時**: `POST /api/analysis/chat` に保持中の `session_id` を含めて送信
- SSE でレスポンスをストリーミング表示（文字が順次表示）
- 「会話をリセット」で新規セッションを開始（現在のセッションは履歴に残る）
- 「この会話を削除」で `DELETE /api/analysis/sessions/{id}` を呼び出し、履歴バーから除去
- Enter キーで送信、Shift+Enter で改行
- SSE の `error` イベント受信時: それまでの返答末尾に「（エラーが発生しました）」を付記し、エラートーストを表示する
- 質問セットドロップダウン: `GET /api/question-sets` で取得、デフォルトを初期選択
- 質問ボタンは常時表示。クリックでその質問文を入力欄にセットして送信
- 質問セットドロップダウン変更時はボタン表示を切り替える（会話はリセットしない）

### S07 設定

**目的**: LLM API キーの登録・確認・削除を行う

**レイアウト**

```
┌────────────────────────────────────────────────┐
│  設定                                          │
│                                                │
│  AI 分析 API                                   │
│  ─────────────────────────────────────────     │
│  利用する API: ● Claude  ○ OpenAI             │
│                                                │
│  API キー                                      │
│  [********************************] [削除]     │
│  ※ 設定済みの場合はマスク表示                  │
│                                                │
│  [新しい API キーを登録する]                   │
│                                                │
│  ✅ API キーを保存しました                      │
│  （または ❌ 無効な API キーです）              │
│                                                │
└────────────────────────────────────────────────┘
```

**仕様**

- `GET /api/settings` で現在の設定を取得（API キーはマスク表示）
- `PUT /api/settings` で API キーを保存
- `DELETE /api/settings/api-key` で API キーを削除
- 保存時に LLM API への疎通確認を行い、結果をフィードバック
- API キーは画面上でマスク表示（`****...****`）

**レイアウト（プロンプトテンプレート編集）**

```
┌────────────────────────────────────────────────┐
│  ← 戻る  プロンプトテンプレートの編集           │
│                                                │
```

**レイアウト（質問セット編集）**

```
┌────────────────────────────────────────────────┐
│  ← 戻る  質問セットの編集                       │
│                                                │
│  セット名                                      │
│  [基本分析                                   ] │
│                                                │
│  質問一覧                        [+ 質問を追加] │
│  ─────────────────────────────────────────     │
│  ［↕］私のプレイの傾向を教えてください  [削除]  │
│  ［↕］先手・後手の勝率の差について...   [削除]  │
│  ［↕］マリガンの影響について...        [削除]  │
│                                                │
│                                   [保存]        │
└────────────────────────────────────────────────┘
```

**仕様**

- `GET /api/prompt-templates` で一覧取得
- `POST /api/prompt-templates` で新規作成
- `PUT /api/prompt-templates/{id}` で更新（デフォルト設定含む）
- `DELETE /api/prompt-templates/{id}` で削除（デフォルトは削除不可、削除ボタンを `--` で非活性表示）
- `GET /api/question-sets` で質問セット一覧取得
- `POST /api/question-sets` で新規作成
- `PUT /api/question-sets/{id}` で更新（質問一覧は全件洗い替え）
- `DELETE /api/question-sets/{id}` で削除（デフォルトは削除不可）
- 質問の並び替えはドラッグ＆ドロップで行い、`display_order` を更新する

## デザインテーマ

**MTGO ライクテーマ（ライト）**

MTGOの古典的なインターフェースに合わせた、羊皮紙・くすみ系の落ち着いたライトテーマ。

### カラーパレット

| 用途 | カラーコード | 説明 |
|------|-------------|------|
| 背景 | `#f0ece0` | 羊皮紙・くすんだオフホワイト |
| サーフェス | `#faf7f0` | カード面・明るいクリーム |
| ボーダー | `#c8b89a` | くすんだベージュ |
| アクセント | `#4a6fa5` | MTGOの青に近い落ち着いた青 |
| テキスト（メイン） | `#2c2416` | 濃い茶色 |
| テキスト（サブ） | `#7a6a55` | 中間の茶色 |
| 成功 | `#5a7a4a` | 落ち着いた緑 |
| 警告 | `#c8622a` | くすんだオレンジ |
| エラー | `#a03030` | くすんだ赤 |

### 方針

- ダークモードは初期リリースでは対応しない（将来の拡張候補）
- レスポンシブ対応は不要（デスクトップ固定 1200x800）
- グラフライブラリ: ECharts のテーマカラーも上記パレットに合わせる
