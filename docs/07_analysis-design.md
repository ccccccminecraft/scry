# 解析・分析設計

## 概要

インポートした MTGO 対戦ログから、統計集計と AI 分析の2段階で知見を抽出する。

---

## 解析の全体像

```
対戦ログ（.dat）
  │
  ▼ パース（MTGOLogParser）
構造化データ → SQLite
  │
  ├──► 統計集計（集計クエリ）→ S05 統計ダッシュボード
  │
  └──► AI チャット（SSE ストリーミング）→ S06 AI 分析（チャット UI）
```

---

## 1. 統計集計

### 集計指標一覧

| 指標 | 説明 | 計算元テーブル |
|------|------|----------------|
| 総マッチ数 | インポート済み対戦の総数 | `matches` |
| 勝率 | 勝利マッチ数 / 総マッチ数 | `matches` |
| ゲーム勝率 | 勝利ゲーム数 / 総ゲーム数 | `games` |
| 平均ターン数 | 全ゲームのターン数の平均 | `games` |
| 先手勝率 | 先手時の勝利ゲーム数 / 先手ゲーム数 | `games` |
| 後手勝率 | 後手時の勝利ゲーム数 / 後手ゲーム数 | `games` |
| マリガン率 | マリガンを1回以上行ったゲーム数 / 総ゲーム数 | `mulligans` |
| マリガン回数別勝率 | マリガン 0回/1回/2回+ ごとの勝率 | `mulligans`, `games` |
| 勝率推移 | 時系列での勝率変化（直近 N マッチ） | `matches` |
| デッキ別勝率 | 使用デッキごとの勝率 | `match_players` |
| ゲームプラン別勝率 | aggro/midrange/control/combo ごとの勝率 | `match_players`, `games` |
| 対戦相手別勝率 | 特定の対戦相手に対する勝率 | `match_players`, `matches` |
| カード別統計 | カードごとのプレイ頻度と登場ゲームの勝率 | `actions`, `games` |

### 集計の粒度

- **全体集計**: 全マッチを対象
- **プレイヤー別集計**: 指定プレイヤーが参加したマッチのみ対象

---

## 2. AI チャット分析

### 目的

統計数値だけでは見えない**プレイパターンの傾向・改善点**について、チャット形式で自然言語で対話する。

### 会話の流れ

```
1. ユーザーがプレイヤー・プロンプトテンプレート・質問セットを選択
      │
      ▼
2. バックエンドが統計データを取得し、選択されたテンプレートの
   {player_name} / {stats_text} を置換してシステムプロンプトを生成
      │
      ▼
3. AI が最初のメッセージを自動送信（データ読み込み完了の挨拶）
   → 新規セッションが自動作成され、session_id がフロントに返る
   → 挨拶メッセージが analysis_messages に保存される
      │
      ▼
4. ユーザーとの自由なチャット（SSE ストリーミング）
   ├── 自由入力 → メッセージをそのまま送信
   └── 質問ボタン → クリックで質問文を送信（常時表示）
   → ユーザー / アシスタント の各メッセージが analysis_messages に順次保存
      │
      ▼
5. 「会話をリセット」または別セッション選択でいつでも過去の会話を参照可能
```

### プロンプトテンプレート

システムプロンプトはユーザーが S08 で作成・編集できる**プロンプトテンプレート**として管理する。`{player_name}` と `{stats_text}` の2つのプレースホルダーは必須で、チャット開始時にバックエンドが実データに置換する。

**デフォルトテンプレート**

```
あなたはMagic: The Gatheringの対戦分析エキスパートです。
以下のプレイヤー統計データをもとに、ユーザーの質問に日本語で答えてください。

回答の方針:
- 統計データに含まれる具体的な数値を根拠として示してください
- 結論 → 根拠 → 改善提案 の順で回答してください
- データにない情報（デッキリスト・相手のプレイング等）については「データがないため不明」と伝えてください
- 回答は簡潔にまとめ、要点を箇条書きで示してください

## 対象プレイヤー
{player_name}

## 統計データ
{stats_text}
```

チャット開始時（プレイヤー選択時）に AI が自動送信する最初のメッセージ（固定文）:

```
{player_name} さんの統計データを読み込みました。何でも聞いてください！
```

### 入力データ（stats_text に埋め込む内容）

```
総マッチ数: 120
使用フォーマット: modern（90件）, unknown（30件）
勝率: 58.3%
先手勝率: 62.1% / 後手勝率: 54.2%
平均ターン数: 9.2
マリガン率: 31.0%
マリガン 0回時勝率: 61.5%
マリガン 1回時勝率: 52.0%
マリガン 2回+時勝率: 38.0%

デッキ別勝率:
  Modern Burn（aggro）: 60試合 61.7%
  Amulet Titan（combo）: 60試合 55.0%

ゲームプラン別勝率:
  aggro: 60試合 61.7%
  combo: 60試合 55.0%

よく使うカード TOP5（登場ゲーム勝率）:
  Lightning Bolt: 143回使用 64.4%
  Goblin Guide: 98回使用 61.2%
  ...
```

デッキ名・ゲームプランが未設定のマッチは対応する集計を省略する。

### LLM 利用方針

| 項目 | 方針 |
|------|------|
| 利用形態 | 外部 LLM API（Claude API または OpenAI API）、SSE ストリーミング |
| 利用モデル | ユーザーが設定画面で選択（デフォルト: Claude） |
| API キー管理 | ユーザーが設定画面で入力・OS キーストア（keyring）に保存 |
| プロンプト管理 | ユーザーが S08 でテンプレートを作成・編集。バックエンドが `{player_name}` / `{stats_text}` を置換 |
| 会話履歴管理 | フロント（Vue）側で保持・管理。バックエンドはステートレス |
| 会話セッション保存 | `analysis_sessions` / `analysis_messages` テーブルに自動保存。過去の会話を S06 で参照可能 |
| キャッシュ | 統計集計クエリは 100〜200 マッチ程度では数十 ms 以内のため、キャッシュなし |
| API キー未設定時 | S06 画面に「APIキーが設定されていません」と表示し、設定画面へ誘導する |
